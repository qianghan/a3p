B · Can't log in? "Database is not available"

The 503 from /auth/login can come from two distinct Neon failure modes — diagnose first, then fix.

Reproduced 2026-05-28: Direct psql against the Neon endpoint returned: 
ERROR: Your account or project has exceeded the compute time quota.
       Upgrade your plan to increase limits.
That's quota exhausted, not a suspended branch. Auto-wake doesn't help here — only an upgrade or waiting for the monthly reset.
Quota reset estimate. The Vercel-Neon integration shows ~61 days old in vercel env ls → project created around 2026-03-28. Neon free-tier compute resets monthly on the cycle anniversary, so the next reset is expected around 2026-05-28 / 2026-05-29 (within ~24 hours).

Confirm the exact moment at console.neon.tech → project rough-dream-72470468 → Billing → Current billing period. The end date is your reset.

If you want to test before then — only the upgrade path (B.1) clears the block. Wait-it-out is free.
B.0a — Poll-and-alert loop (run in spare terminal while you wait) Copy/Paste
Direct probe every 10 min, beeps when the quota clears. Safer than refreshing the production URL by hand. 
brew install libpq 2>/dev/null
export PATH="/opt/homebrew/opt/libpq/bin:$PATH"

while true; do
  vercel env pull /tmp/.env.prod --environment production --yes >/dev/null 2>&1
  DBURL=$(grep '^a3p_DATABASE_URL_UNPOOLED=' /tmp/.env.prod | cut -d= -f2- | tr -d '"')
  if psql "$DBURL" -c "SELECT 1;" >/dev/null 2>&1; then
    echo "🟢 $(date) — Neon is BACK. You can log in now."
    osascript -e 'display notification "Neon quota reset — DB is up" with title "AgentBook"' 2>/dev/null
    say "Neon is back" 2>/dev/null
    rm /tmp/.env.prod
    break
  fi
  echo "🔴 $(date) — still blocked. Sleeping 10 min."
  rm /tmp/.env.prod
  sleep 600
done
